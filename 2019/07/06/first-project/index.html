新手项目总结

项目基本要求

搭建Gradle工程，构建一个RESTful web 服务，完成下功能：
商品列表页展示
商品详情展示
用户认证 （需考虑安全性问题）
订单创建
订单支付（无需真实接入第三方支付）
订单详细展示
订单列表展示
编写单元测试，Cover所有的路径

坑点细数

首先学习使用 Gradle+Spring 搭建项目

利用Gradle搭建工程一般可以通过两种途径
1）Intellij直接创建工程,注意Gradle版本
2) 使用官网模板 本地build再导入

每个文档文件目录可以设置为resoutces或者main等

如果文件目录对但是文档不对，可能导致当前目录下的资源扫描失败，例如resources文件未被标注为resources，其目录下的配置文件就可能无法扫描，导致注入失败。

对于整个工程，要按照功能划分模块

基本分为BasicStruct层，API层，Services层，DAO层，Controller层.

BasicStruct层

数据类型要implements Serializable 之后在利用dubbo rpc调用数据库查询时不序列化会出现问题

API层

用Interface实现的接口层，在用dubbo部署之后上述两层应处于公共层，在服务提供方和消费方共享

Services层

服务层 实现接口层的方法 并且负责和DAO交互 也是Provider的主力

DAO层

Mybatis访问数据库的实现层，供Services层调用

Contoller 层

各种业务逻辑的具体层 应单独处于Consumer层

整个项目踩过的坑以及解决方法

1.Mybatis配置

2.利用Dubbo拆分服务

3.接入CAT监控
